국방 로봇 운용의 표준을 다시 설계합니다

P-Fleet은 VDA 5050 기반 이기종 로봇 통합 관제 플랫폼입니다. 제조사별로 분절된 국방 로봇을 하나의 Fleet으로 연결하고, AI Orchestration을 통해 실전적인 배치·통제·가시성을 제공합니다.

핵심 메시지

개별 로봇 제어가 아니라, 복수 에이전트와 복수 하드웨어를 하나의 작전 체계로 묶는 관제 레이어가 필요합니다.

왜 지금인가

국방 로봇 도입은 늘고 있지만, 실제 현장에서는 벤더 종속과 보안 문제 때문에 통합 운용이 가장 큰 병목으로 남아 있습니다.

VDA 5050
하드웨어 중립형 통합을 위한 핵심 표준
표준 기반 접근으로 벤더 락인을 줄이고, 장기적으로는 국방 로봇 스택의 상위 오케스트레이션 레이어를 지향합니다.

2026년 목표

MVP 고도화, 자체 테스트베드 구축, Fleet-Level AI 검증, Gazebo 시나리오 실증

P-Fleet의 답: Orchestration Layer
Investment Thesis

왜 P-Fleet인가

로봇 하드웨어가 늘어날수록, 결국 가치는 “개별 기체”보다 “운용 체계”에서 커집니다. P-Fleet은 그 운영 체계를 표준화하고, AI로 고도화하는 팀입니다.
01

Vendor Lock-in 해소

특정 제조사 중심의 폐쇄형 운용 구조에서 벗어나, 서로 다른 로봇을 동일한 프로토콜 아래에서 통합 운용할 수 있게 합니다.

02

국방용 보안 설계

폐쇄망 환경과 명령 무결성, 접근제어를 고려한 설계를 통해 민수용 관제 솔루션과 다른 국방형 아키텍처를 제안합니다.

03

Fleet-Level AI

단일 로봇의 자율성보다 상위 레벨의 배치, 작업 할당, 우선순위 조정이 더 큰 운용 효율을 만들어냅니다.

04

Software Margin 구조

하드웨어 제조가 아닌 소프트웨어 레이어에서 반복 가능한 라이선스, 유지보수, 컨설팅 수익 모델을 구축합니다.

Problem

문제는 로봇이 아니라
운용의 분절입니다

현재 국방 로봇 도입은 장비 단위 구매 중심으로 이루어져 있습니다. 결과적으로 로봇은 늘어나도, 전체 작전을 하나로 묶는 관제·배치·협업 체계는 부재합니다.

하드웨어 종속

제조사마다 상이한 인터페이스와 운영 방식으로 인해 부대 단위 유연 편제와 조합형 운용이 어렵습니다.

  • 장비 추가 시 통합 비용 증가
  • 대체재 확보 어려움
  • 현장 운영자 학습비용 증가

실전 환경의 비결정성

작전 환경은 예측 불가합니다. 임무 우선순위, 통신 상태, 장비 가용성이 계속 바뀌는 상황에서 정적 제어만으로는 부족합니다.

  • 동적 태스크 재분배 필요
  • 로봇 상태 기반 실시간 판단 필요
  • 복수 에이전트 간 컨텍스트 전달 필요

보안형 통합관제 공백

민수형 플랫폼을 그대로 전용하기 어려운 이유는 명확합니다. 폐쇄망, 무결성, 접근제어, 감사가능성까지 국방형 요구조건이 존재하기 때문입니다.

  • M-BcN 연동 안정성
  • HMAC 기반 명령 검증
  • CC 인증 기준 고려
Solution

P-Fleet은 상위 운영체계입니다

P-Fleet은 개별 로봇을 제어하는 앱이 아니라, 다양한 로봇과 임무를 하나의 운영 계층에서 조정하는 Orchestration Platform입니다.

무엇을 하는가

  • VDA 5050 기반의 하드웨어 중립형 연결 레이어
  • 임무·상태·가용성 기반의 Fleet-Level AI 할당
  • ROS2/MQTT 기반 실시간 워크플로 제어
  • 운영자용 실시간 대시보드 및 관제 인터페이스

어떤 가치를 만드는가

  • 도입 장비 종류가 늘어날수록 플랫폼 가치 상승
  • 장비 교체/증설에 대한 비용과 위험 감소
  • 작전 중 예외 상황에 대한 재배치 속도 향상
  • 지휘관에게 필요한 가시성과 추적성 제공

Core Architecture

표준, AI, 보안을
하나로 묶습니다

핵심은 세 가지입니다. 다양한 로봇을 연결하는 표준, 복수 자산을 배치하는 AI, 그리고 국방 환경에서 신뢰 가능한 운영을 위한 보안 아키텍처입니다.

Input Layer

이기종 로봇, 센서, 임무 정보, 운영자 명령, 통신 상태, 텔레메트리 데이터가 유입됩니다.

  • UGV / 드론 / 기타 무인체계
  • 상태 데이터 수집
  • 임무 우선순위 입력

P-Fleet Orchestration Layer

표준 인터페이스, Agent allocation, Workflow control, Policy enforcement, Observability를 통합한 상위 운영 계층

Output Layer

실시간 작업 할당, 재배치, 모니터링, 경보, 이력 추적 및 관제 화면으로 연결됩니다.

  • Task dispatch
  • Dashboard & alerts
  • 감사 가능한 로그 축적

Orchestration Layer

단순 경로 제어가 아니라 다중 로봇 간 Context handoff와 Task progress를 관리하는 상위 운영 로직입니다. 실시간 이벤트에 따라 워크플로를 조정합니다.

Visibility & Observability

작전 중 발생하는 상태 변화와 의사결정 경로를 기록·시각화합니다. 지휘관은 실행 결과뿐 아니라 과정까지 추적할 수 있어야 합니다.

Governance & Security

M-BcN 연동, HMAC 기반 무결성 검증, 접근 권한 통제 등 군납형 요구사항을 고려한 보안 설계를 적용합니다.

Business Model

반복 가능한 소프트웨어 수익 구조

제조업 기반 단발성 납품이 아니라, 플랫폼 라이선스와 운영 데이터 기반 부가 수익을 쌓아가는 구조를 지향합니다.

SaaS License

단말 혹은 운용 단위 기준 라이선스 과금 모델입니다. 장비 수 증가와 운영 범위 확장이 매출 레버리지로 연결됩니다.

  • 반복 매출 기반
  • 업데이트/기능 고도화 반영 용이
  • 플랫폼 락인 형성 가능

Predictive Maintenance

텔레메트리 데이터 분석을 통해 이상징후 탐지와 고장 예측 인사이트를 제공합니다. 운용 효율과 가동률 개선이 핵심 가치입니다.

  • 실시간 상태 모니터링
  • 정비 시점 최적화
  • 추가 소프트웨어 매출 가능

MRO & Consulting

기존 제조사와 기관이 표준 기반 운용으로 전환할 수 있도록 시스템 통합, 컨설팅, 유지보수 서비스를 제공합니다.

  • 표준 도입 지원
  • 현장 맞춤 SI
  • 장기 유지보수 계약화 가능
Market Narrative

시장은 하드웨어에서
운영 소프트웨어로 이동합니다

여기서는 아직 구체 수치 대신 투자 스토리 중심으로 정리했습니다. 실제 공개 전에는 TAM/SAM/SOM, 조달 규모, 파일럿 대상 시장 데이터를 추가하면 됩니다.

시장 변화

국방 무인체계는 기체 수보다 운용 복잡도가 더 빠르게 증가합니다. 복수 자산이 동시에 투입될수록 통합 관제와 배치 알고리즘의 가치가 커집니다.

P-Fleet의 포지션

P-Fleet은 제조사가 아닌 상위 소프트웨어 계층에 위치합니다. 이는 특정 하드웨어의 성공 여부와 무관하게, 전체 생태계 확장에 따라 가치가 커질 수 있는 위치입니다.
